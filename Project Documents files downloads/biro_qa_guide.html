🔴 Must Test First

1
Start a round & get a code Core multiplayer flow
Critical
Open the app and go to Setup
Check: there's a Round Name field at the top — does it show today's date + "Westwood" as placeholder?
Optionally type a name like "Biro Test Round"
Set some players and tap Start Round
ExpectedA green banner appears on the Live screen saying something like "🟢 Live — Share code 2847 so others can join"
ExpectedThe 4-digit code is visible. Note it down.
If it failsScreenshot the error or describe what you see instead
2
Join round from another device The whole point of the app
Critical
Open the app on a second device (or ask Tim to start a round and give you the code)
Tap Join in the top nav
Enter the 4-digit code and tap Join
ExpectedYou see the live results screen — same players and scores as the host
ExpectedA "🟢 Live — updates automatically" badge is visible
Common failure"Round not found" even with correct code — note the exact code you used
3
Live score sync Enter a score, does the joiner see it?
Critical
Device 1 (host): enter scores for Hole 1 and tap Save Hole
Device 2 (you): watch the results screen
ExpectedWithin a few seconds, the scores and money results update on Device 2 automatically — no refresh needed
ExpectedThe "Updated" timestamp in the header changes
If it doesn't updateNote how long you waited. Try pulling down to refresh — does it then show?

🟡 Scoring & Money

4
Money adds up to zero No one creates or loses money out of thin air
Important
Enter scores for a full round (or at least 9 holes)
Go to Results screen and look at the Projected Settlement
Add up all the dollar amounts for each player
ExpectedAll amounts sum to $0 — what one player wins, another loses. No phantom money.
If it's offTake a screenshot and note the exact amounts shown for each player
5
Handicap strokes look right Does the weaker player get dots on the right holes?
Important
Set up a 2-player match: one player at hcp 0, one at hcp 18
Enter a score, open the Edit Scorecard view
ExpectedThe 18-hcp player shows a dot (•) after their score on every hole
ExpectedA tie on gross score should count as a net win for the higher-hcp player on holes where they get a stroke
6
Birdie side bets pay correctly Birdie: winners get paid, everyone else splits the cost
Important
Enable Birdies in Setup with a $5 bet
Enter a birdie score (score = par − 1) for one player on a hole
ExpectedBirdie section shows that player winning, others paying
ExpectedWith 5 players: birdie wins $20, other 4 each pay $5 — total still nets $0

🟢 Polish & UX

7
Round name auto-fills sensibly Leave the name blank, start round
Polish
Go to Setup — leave the Round Name field completely empty
Start the round
ExpectedThe name auto-fills to something like "May 15 - Westwood"
8
App works on your phone Layout, buttons, score entry
Polish
Run through the full flow on your phone (not desktop)
Check: score entry buttons are big enough to tap
Check: nothing overflows or gets cut off sideways
ExpectedEverything usable with thumbs. Nothing looks broken.
9
Auto-restore after closing the app Close tab, reopen — are you back where you were?
Nice to have
Start a round, enter scores for a few holes
Close the browser tab completely
Reopen golf-app-neon.vercel.app
ExpectedYour scores and setup are still there — app picks up where you left off
